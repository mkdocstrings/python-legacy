27 default_config: dict = {"filters": ["!^_[^_]"]} 

28 """The default selection options. 

29 

30 Option | Type | Description | Default 

31 ------ | ---- | ----------- | ------- 

32 **`filters`** | `List[str]` | Filter members with regular expressions. | `[ "!^_[^_]" ]` 

33 **`members`** | `Union[bool, List[str]]` | Explicitly select the object members. | *`pytkdocs` default: `True`* 

34 

35 If `members` is a list of names, filters are applied only on the members children (not the members themselves). 

36 If `members` is `False`, none are selected. 

37 If `members` is `True` or an empty list, filters are applied on all members and their children. 

38 

39 Members affect only the first layer of objects, while filters affect the whole object-tree recursively. 

40 

41 Every filters is run against every object name. An object can be un-selected by a filter and re-selected by the 

42 next one: 

43 

44 - `"!^_"`: exclude all objects starting with an underscore 

45 - `"^__"`: but select all objects starting with **two** underscores 

46 

47 Obviously one could use a single filter instead: `"!^_[^_]"`, which is the default. 

48 """

54 

55 When trying to fix (optional) cross-references, the autorefs plugin will try to collect 

56 an object with every configured handler until one succeeds. It will then try to get

106 def collect(self, identifier: str, config: dict) -> CollectorItem: 

107 """Collect the documentation tree given an identifier and selection options. 

108 

109 In this method, we feed one line of JSON to the standard input of the subprocess that was opened 

110 during instantiation of the collector. Then we read one line of JSON on its standard output. 

111 

112 We load back the JSON text into a Python dictionary. 

113 If there is a decoding error, we log it as error and raise a CollectionError. 

114 

115 If the dictionary contains an `error` key, we log it as error (with the optional `traceback` value), 

116 and raise a CollectionError. 

117 

118 If the dictionary values for keys `loading_errors` and `parsing_errors` are not empty, 

119 we log them as warnings. 

120 

121 Then we pick up the only object within the `objects` list (there's always only one, because we collect 

122 them one by one), rebuild it's categories lists 

123 (see [`rebuild_category_lists()`][mkdocstrings_handlers.python.collector.rebuild_category_lists]), 

124 and return it. 

125 

126 Arguments: 

127 identifier: The dotted-path of a Python object available in the Python path. 

128 config: Selection options, used to alter the data collection done by `pytkdocs`. 

129 

130 Raises: 

131 CollectionError: When there was a problem collecting the object documentation. 

132 

133 Returns: 

134 The collected object-tree. 

135 """ 

136 final_config = ChainMap(config, self.default_config) 

137 

138 log.debug("Preparing input") 

139 json_input = json.dumps({"objects": [{"path": identifier, **final_config}]}) 

140 

141 log.debug("Writing to process' stdin") 

142 self.process.stdin.write(json_input + "\n") # type: ignore 

143 self.process.stdin.flush() # type: ignore 

144 

145 log.debug("Reading process' stdout") 

146 stdout = self.process.stdout.readline() # type: ignore 

147 

148 log.debug("Loading JSON output as Python object") 

149 try: 

150 result = json.loads(stdout) 

151 except json.decoder.JSONDecodeError as exception: 

152 error = "\n".join(("Error while loading JSON:", stdout, traceback.format_exc())) 

153 raise CollectionError(error) from exception 

154 

155 if "error" in result: 

156 error = result["error"] 

157 if "traceback" in result: 

158 error += f"\n{result['traceback']}" 

159 raise CollectionError(error)

182def rebuild_category_lists(obj: dict) -> None: 

183 """Recursively rebuild the category lists of a collected object. 

184 

185 Since `pytkdocs` dumps JSON on standard output, it must serialize the object-tree and flatten it to reduce data 

186 duplication and avoid cycle-references. Indeed, each node of the object-tree has a `children` list, containing 

187 all children, and another list for each category of children: `attributes`, `classes`, `functions`, `methods` 

188 and `modules`. It replaces the values in category lists with only the paths of the objects. 

189 

190 Here, we reconstruct these category lists by picking objects in the `children` list using their path. 

191 

192 For each object, we recurse on every one of its children. 

193 

194 Arguments: 

195 obj: The collected object, loaded back from JSON into a Python dictionary. 

196 """ 

197 for category in ("attributes", "classes", "functions", "methods", "modules"): 

198 obj[category] = [obj["children"][path] for path in obj[category]] 

199 obj["children"] = [child for _, child in obj["children"].items()] 

200 for child in obj["children"]: 

201 rebuild_category_lists(child)
